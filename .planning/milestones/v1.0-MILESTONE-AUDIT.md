## Cross-Phase Integration

**Score: 30/33 requirement integrations verified**

### Integration Issues

**INT-01 (Medium): `updateProfile` Server Action orphaned**
- `src/lib/actions/profile.ts` exports `updateProfile` (line 78) but no component imports it
- Settings page at `/settings` displays profile data read-only with no edit controls
- Users cannot modify trading name, working hours, or slot size after completing onboarding
- Affects: AUTH-02, AUTH-03, AUTH-04 (post-onboarding edit capability — requirements are satisfied for initial setup via onboarding, but edit-after-setup is missing)

**INT-02 (Low): Login redirect through `/onboarding`**
- `login/actions.ts` always redirects to `/onboarding` after successful login
- Returning users get an extra server round-trip (`/onboarding` checks `onboarding_completed` then redirects to `/diary`)
- Works correctly but adds unnecessary latency for returning users
- Affects: AUTH-01 (functional, not broken)

**INT-03 (Low): Timezone hardcoded to Europe/London**
- `diary-view.tsx` accepts `timezone` prop but never uses it
- `utils.ts` hardcodes `'Europe/London'` in `formatTime` and `formatDiaryDate`
- Could cause 1-hour display errors during BST/GMT transitions if server/client disagree
- Affects: BOOK-01, BOOK-02 (display accuracy edge case)
