    - `initialDate: Date` ŌĆö the date/time to book (from slot tap or FAB)

    - `slotMinutes: number` ŌĆö for end time calculation
    - `onBooked: () => void` ŌĆö callback to refresh diary after successful booking

    Internal state machine ŌĆö 3 steps:
    - `step: 'client' | 'services' | 'confirm'`
    - `selectedClient: Client | null`
    - `selectedServices: Service[]` (multi-select)
    - `bookingNotes: string`
    - `overrideHours: boolean`

    **Step 1: Select Client**
    - Search input at top with debounced text (300ms delay using setTimeout/clearTimeout pattern)
    - Call `searchClients(query)` on debounced input change
    - Display results in BottomSheetFlatList (from @gorhom/bottom-sheet ŌĆö required for scroll inside sheet)
    - Each row: client name, phone (if present)
    - Tap selects client, advances to step 2
    - Show ClientNotesPreview below the selected client name (after selection, before advancing ŌĆö or inline in step 2)

    **Step 2: Select Services**
    - Load services using `getServices()` on step enter
    - Group by category (use the service_categories join or build from data)
    - SectionList or grouped FlatList: category headers, service rows
    - Each row: service name, duration, price (formatPennies), checkbox/toggle for selection
    - Multi-select: tap toggles selection
    - Running totals bar at bottom:
      - "N services" count
      - Total duration: sum of selected service durations, displayed as "Xh Ym" or "X min"
      - Total price: sum of selected service prices, formatPennies
    - "Next" button (disabled if no services selected)
    - Show ClientNotesPreview compact card above the service list for context

    **Step 3: Confirm & Book**
    - Summary display:
      - Client name
      - Date: formatted from initialDate
      - Start time: initialTime or default
      - End time: auto-calculated as start + sum of service durations
      - Services list with individual prices
      - Total price
    - Notes input: TextInput for optional booking notes
    - Working hours validation:
      - Call `isWithinWorkingHours(startsAt, endsAt, workingHours)` pre-check
      - If invalid: show warning banner with reason text and "Book anyway" toggle (sets overrideHours)
      - If valid: no warning shown
    - "Confirm Booking" button:
      - Calls `createAppointment(input)` with all collected data
      - Construct starts_at and ends_at as ISO strings from date + time
      - Services array: map selectedServices to snapshot format `{ service_id, service_name, service_price, service_duration_minutes }`
      - Handle response:
        - `{ warning }`: show Alert with warning text and "Book anyway" / "Cancel" options. "Book anyway" re-calls with override_working_hours=true.
        - `{ error }`: show Alert with error message (includes double-booking message)
        - `{ success }`: dismiss sheet, call onBooked callback, show success feedback
      - Loading state on button during API call

    Sheet configuration:
    - snapPoints: ['85%']
    - enablePanDownToClose: true
    - backgroundStyle: theme.background
    - handleIndicatorStyle: theme.border
    - Back button in header to go to previous step (step 2 -> step 1, step 3 -> step 2)
    - Step indicator dots or "Step X of 3" text

    Use BottomSheetScrollView or BottomSheetFlatList for scrollable content inside the sheet (critical: regular ScrollView/FlatList won't work inside BottomSheetModal).
