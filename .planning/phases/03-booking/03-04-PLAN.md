<task type="checkpoint:human-verify" gate="blocking">
  <name>Verify Phase 3: Booking — complete diary and booking flow</name>
  <what-built>
    Complete diary and booking system:
    - Slot-based diary page with CSS Grid layout and time labels
    - Date navigation (prev/next day buttons, native date picker)
    - Scroll-to-current-time on today's diary
    - Current time red line indicator
    - Multi-step booking sheet (select client → select services → confirm with notes)
    - Working hours validation with override option
    - Double-booking prevention (PostgreSQL exclusion constraint)
    - Appointment detail sheet with status changes (complete, cancel, no-show)
    - Appointment reschedule with new date/time
    - Floating + button for quick booking access
  </what-built>
  <how-to-verify>
    Start the dev server: `npm run dev`
    Open http://localhost:3000/diary on your phone or mobile simulator

    **1. Diary View (BOOK-01)**
    - [ ] Diary shows today's date with slot-based grid
    - [ ] Time labels appear on hour boundaries in the left gutter
    - [ ] Current time red line is visible at the correct position
    - [ ] Page scrolls to current time on load

    **2. Date Navigation (BOOK-02)**
    - [ ] Tap left chevron → navigates to yesterday
    - [ ] Tap right chevron → navigates to tomorrow
    - [ ] Tap date text → opens native date picker
    - [ ] Select a date → diary shows that day

    **3. Create Appointment (BOOK-03, BOOK-04)**
    - [ ] Tap an empty time slot → booking sheet opens
    - [ ] Step 1: Search for a client by name → results appear
    - [ ] Tap a client → advances to Step 2
    - [ ] Step 2: Select one or more services → totals update
    - [ ] Tap Continue → advances to Step 3
    - [ ] Step 3: Summary shows client, time, services, total
    - [ ] Add optional notes in the textarea
    - [ ] Tap "Book Appointment" → appointment appears in diary grid
    - [ ] Appointment block is positioned at the correct time with correct height

    **4. Double-Booking Prevention (BOOK-07)**
    - [ ] Try to book another appointment at the SAME time → error message shown
    - [ ] Error is user-friendly (not a raw Postgres error)
    - [ ] The booking sheet stays open (not dismissed) so user can choose a different time

    **5. Appointment Management (BOOK-05)**
    - [ ] Tap an existing appointment block → detail sheet opens
    - [ ] Detail sheet shows client name, services, time, notes, status
    - [ ] Tap "Complete" → status changes to completed, block turns green
    - [ ] Tap "Cancel" → confirmation prompt → status changes, block is greyed out
    - [ ] Status badge updates correctly

    **6. Reschedule (BOOK-06)**
    - [ ] From appointment detail sheet, tap "Reschedule"
    - [ ] Change the date and/or time
    - [ ] Tap "Confirm Reschedule" → appointment moves to new position

    **7. Working Hours (BOOK-08)**
    - [ ] Book an appointment outside working hours (e.g. 7am if hours start at 9am)
    - [ ] Warning message appears about being outside working hours
    - [ ] Check the override checkbox → booking proceeds successfully

    **8. General UX**
    - [ ] Bottom nav diary tab is highlighted when on /diary
    - [ ] All sheets slide up smoothly from the bottom
    - [ ] Loading states shown during submissions
    - [ ] Floating + button is visible and opens booking sheet
