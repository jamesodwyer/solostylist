In `src/app/layout.tsx`, add `interactiveWidget: "resizes-content"` to the existing `viewport` export.

The viewport export should become:

```typescript
export const viewport: Viewport = {
  width: "device-width",
  initialScale: 1,
  maximumScale: 1,
  viewportFit: "cover",
  themeColor: "#000000",
  interactiveWidget: "resizes-content",
}
```

This tells iOS Safari (16+) and Chrome (108+) to shrink the layout viewport when the keyboard opens, which fixes the scroll-trap issue in bottom sheets with text inputs (BookingSheet notes field, PaymentSheet amount field, etc.).

If TypeScript reports an error on the `interactiveWidget` field (unlikely with Next.js 16, but possible), suppress with `// @ts-expect-error interactiveWidget supported in Next.js 14.1+` on the line above it. Do NOT cast the entire object.
