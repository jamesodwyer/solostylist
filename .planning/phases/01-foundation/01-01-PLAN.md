<task type="auto">
  <name>Task 2: Create database migration with all 12 tables, RLS, and btree_gist</name>
  <files>
    supabase/migrations/20260301000000_initial_schema.sql
  </files>
  <action>
    Create the SQL migration file at `supabase/migrations/20260301000000_initial_schema.sql`. This single migration defines the entire schema. All tables use UUID primary keys, `owner_user_id UUID NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE`, and `TIMESTAMPTZ` for all timestamps with `DEFAULT now()`.

    **All monetary values are INTEGER (pennies), never float/decimal** — per PROJECT.md decision.

    Enable extensions first:
    ```sql
    CREATE EXTENSION IF NOT EXISTS btree_gist;
    ```

    **The 12 tables (in dependency order):**

    1. **profiles** — Business profile (one per user)
       - `id UUID PRIMARY KEY REFERENCES auth.users(id) ON DELETE CASCADE` (same as auth user ID, NOT gen_random_uuid)
       - `owner_user_id UUID NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE`
       - `trading_name TEXT`
       - `phone TEXT`
       - `address TEXT`
       - `default_slot_minutes INTEGER NOT NULL DEFAULT 15`
       - `working_hours JSONB NOT NULL DEFAULT '{}'::jsonb` (keys: "monday".."sunday", values: {start, end} or null for day off)
       - `timezone TEXT NOT NULL DEFAULT 'Europe/London'`
       - `onboarding_completed BOOLEAN NOT NULL DEFAULT false`
       - `created_at TIMESTAMPTZ NOT NULL DEFAULT now()`
       - `updated_at TIMESTAMPTZ NOT NULL DEFAULT now()`
       - UNIQUE constraint on `owner_user_id` (one profile per user)

    2. **service_categories** — Service groupings
       - `id UUID PRIMARY KEY DEFAULT gen_random_uuid()`
       - `owner_user_id UUID NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE`
       - `name TEXT NOT NULL`
       - `sort_order INTEGER NOT NULL DEFAULT 0`
       - `created_at TIMESTAMPTZ NOT NULL DEFAULT now()`
       - `updated_at TIMESTAMPTZ NOT NULL DEFAULT now()`

    3. **services** — Service catalogue
       - `id UUID PRIMARY KEY DEFAULT gen_random_uuid()`
       - `owner_user_id UUID NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE`
       - `category_id UUID REFERENCES public.service_categories(id) ON DELETE SET NULL`
       - `name TEXT NOT NULL`
       - `duration_minutes INTEGER NOT NULL`
       - `price INTEGER NOT NULL` (pennies)
       - `is_active BOOLEAN NOT NULL DEFAULT true`
       - `deposit_type TEXT CHECK (deposit_type IN ('none', 'fixed', 'percentage'))` DEFAULT 'none'
       - `deposit_value INTEGER NOT NULL DEFAULT 0` (pennies for fixed, percentage points for percentage)
       - `deposit_required BOOLEAN NOT NULL DEFAULT false`
       - `created_at TIMESTAMPTZ NOT NULL DEFAULT now()`
       - `updated_at TIMESTAMPTZ NOT NULL DEFAULT now()`

    4. **clients** — Client records
       - `id UUID PRIMARY KEY DEFAULT gen_random_uuid()`
       - `owner_user_id UUID NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE`
       - `first_name TEXT NOT NULL`
       - `last_name TEXT`
       - `phone TEXT`
       - `email TEXT`
       - `address TEXT`
       - `marketing_consent BOOLEAN NOT NULL DEFAULT false`
       - `created_at TIMESTAMPTZ NOT NULL DEFAULT now()`
       - `updated_at TIMESTAMPTZ NOT NULL DEFAULT now()`

    5. **client_notes** — General notes, colour formulas, treatment records
       - `id UUID PRIMARY KEY DEFAULT gen_random_uuid()`
       - `owner_user_id UUID NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE`
       - `client_id UUID NOT NULL REFERENCES public.clients(id) ON DELETE CASCADE`
       - `note_type TEXT NOT NULL CHECK (note_type IN ('general', 'colour_formula', 'treatment'))`
       - `content TEXT NOT NULL`
       - `created_at TIMESTAMPTZ NOT NULL DEFAULT now()`
       - `updated_at TIMESTAMPTZ NOT NULL DEFAULT now()`

    6. **colour_formulas** — Dedicated colour formula records (separate from notes for structured data)
       - `id UUID PRIMARY KEY DEFAULT gen_random_uuid()`
       - `owner_user_id UUID NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE`
       - `client_id UUID NOT NULL REFERENCES public.clients(id) ON DELETE CASCADE`
       - `formula TEXT NOT NULL`
       - `notes TEXT`
       - `created_at TIMESTAMPTZ NOT NULL DEFAULT now()`
       - `updated_at TIMESTAMPTZ NOT NULL DEFAULT now()`

    7. **tags** — Tag definitions
       - `id UUID PRIMARY KEY DEFAULT gen_random_uuid()`
       - `owner_user_id UUID NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE`
       - `name TEXT NOT NULL`
       - `created_at TIMESTAMPTZ NOT NULL DEFAULT now()`
       - UNIQUE constraint on `(owner_user_id, name)` to prevent duplicate tag names per user

    8. **client_tags** — Junction table
       - `id UUID PRIMARY KEY DEFAULT gen_random_uuid()`
       - `owner_user_id UUID NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE`
       - `client_id UUID NOT NULL REFERENCES public.clients(id) ON DELETE CASCADE`
       - `tag_id UUID NOT NULL REFERENCES public.tags(id) ON DELETE CASCADE`
       - `created_at TIMESTAMPTZ NOT NULL DEFAULT now()`
       - UNIQUE constraint on `(client_id, tag_id)`

    9. **appointments** — Booking records
       - `id UUID PRIMARY KEY DEFAULT gen_random_uuid()`
       - `owner_user_id UUID NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE`
       - `client_id UUID NOT NULL REFERENCES public.clients(id) ON DELETE CASCADE`
       - `starts_at TIMESTAMPTZ NOT NULL`
       - `ends_at TIMESTAMPTZ NOT NULL`
       - `status TEXT NOT NULL DEFAULT 'booked' CHECK (status IN ('booked', 'completed', 'cancelled', 'no_show'))`
       - `notes TEXT`
       - `created_at TIMESTAMPTZ NOT NULL DEFAULT now()`
       - `updated_at TIMESTAMPTZ NOT NULL DEFAULT now()`
       - **Exclusion constraint** (btree_gist): Prevent overlapping appointments for the same user where status is 'booked'. Use `EXCLUDE USING gist (owner_user_id WITH =, tstzrange(starts_at, ends_at) WITH &&) WHERE (status = 'booked')`

    10. **appointment_services** — Junction: appointment to services (multi-service bookings)
        - `id UUID PRIMARY KEY DEFAULT gen_random_uuid()`
        - `owner_user_id UUID NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE`
        - `appointment_id UUID NOT NULL REFERENCES public.appointments(id) ON DELETE CASCADE`
        - `service_id UUID NOT NULL REFERENCES public.services(id) ON DELETE RESTRICT`
        - `service_name TEXT NOT NULL` (snapshot at booking time)
        - `service_price INTEGER NOT NULL` (snapshot in pennies)
        - `service_duration_minutes INTEGER NOT NULL` (snapshot)
        - `created_at TIMESTAMPTZ NOT NULL DEFAULT now()`

    11. **payments** — Payment records
        - `id UUID PRIMARY KEY DEFAULT gen_random_uuid()`
        - `owner_user_id UUID NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE`
        - `appointment_id UUID REFERENCES public.appointments(id) ON DELETE SET NULL`
        - `client_id UUID NOT NULL REFERENCES public.clients(id) ON DELETE CASCADE`
        - `amount INTEGER NOT NULL` (pennies)
        - `method TEXT NOT NULL CHECK (method IN ('cash', 'card'))`
        - `payment_type TEXT NOT NULL DEFAULT 'payment' CHECK (payment_type IN ('payment', 'refund', 'void'))`
        - `reference_payment_id UUID REFERENCES public.payments(id) ON DELETE SET NULL` (for refunds/voids, points to original)
        - `notes TEXT`
        - `paid_at TIMESTAMPTZ NOT NULL DEFAULT now()`
        - `created_at TIMESTAMPTZ NOT NULL DEFAULT now()`

    12. **audit_log** — Immutable audit trail
        - `id UUID PRIMARY KEY DEFAULT gen_random_uuid()`
        - `owner_user_id UUID NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE`
        - `action TEXT NOT NULL` (e.g., 'payment.refund', 'appointment.delete', 'payment.void')
        - `entity_type TEXT NOT NULL` (e.g., 'payment', 'appointment')
        - `entity_id UUID NOT NULL`
        - `details JSONB`
        - `created_at TIMESTAMPTZ NOT NULL DEFAULT now()`
        - **RLS for audit_log:** SELECT and INSERT only (no UPDATE or DELETE policies — audit log is append-only)

    **RLS policies for each table** (except audit_log which gets only SELECT + INSERT):
    Use the optimised `(select auth.uid())` subquery form for all policies:
    ```sql
    ALTER TABLE public.{table} ENABLE ROW LEVEL SECURITY;

    CREATE POLICY "{table}_select_own" ON public.{table}
      FOR SELECT TO authenticated
      USING ((select auth.uid()) = owner_user_id);

    CREATE POLICY "{table}_insert_own" ON public.{table}
      FOR INSERT TO authenticated
      WITH CHECK ((select auth.uid()) = owner_user_id);

    CREATE POLICY "{table}_update_own" ON public.{table}
      FOR UPDATE TO authenticated
      USING ((select auth.uid()) = owner_user_id)
      WITH CHECK ((select auth.uid()) = owner_user_id);

    CREATE POLICY "{table}_delete_own" ON public.{table}
      FOR DELETE TO authenticated
      USING ((select auth.uid()) = owner_user_id);
    ```

    **Profile auto-creation trigger:**
    ```sql
    CREATE OR REPLACE FUNCTION public.handle_new_user()
    RETURNS TRIGGER AS $$
    BEGIN
      INSERT INTO public.profiles (id, owner_user_id, onboarding_completed)
      VALUES (NEW.id, NEW.id, false)
      ON CONFLICT (id) DO NOTHING;
      RETURN NEW;
    END;
    $$ LANGUAGE plpgsql SECURITY DEFINER;

    CREATE TRIGGER on_auth_user_created
      AFTER INSERT ON auth.users
      FOR EACH ROW EXECUTE PROCEDURE public.handle_new_user();
    ```

    **Indexes** to add for performance (queries identified from requirements):
    - `CREATE INDEX idx_clients_owner_name ON public.clients (owner_user_id, first_name, last_name);` (client search)
    - `CREATE INDEX idx_clients_owner_phone ON public.clients (owner_user_id, phone);` (client search by phone)
    - `CREATE INDEX idx_appointments_owner_date ON public.appointments (owner_user_id, starts_at);` (diary view)
    - `CREATE INDEX idx_payments_owner_date ON public.payments (owner_user_id, paid_at);` (daily totals)
    - `CREATE INDEX idx_payments_appointment ON public.payments (appointment_id);` (payment lookup by appointment)
    - `CREATE INDEX idx_client_notes_client ON public.client_notes (client_id);` (notes per client)
    - `CREATE INDEX idx_appointment_services_appointment ON public.appointment_services (appointment_id);` (services per appointment)
  </action>
