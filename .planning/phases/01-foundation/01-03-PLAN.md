    **3. Install banner (`src/components/install-banner.tsx`):**

    Client component implementing the PWA install prompt per user decisions:

    **Trigger:** Show after first successful sign-in (when component mounts in authenticated app shell).

    **Dismiss logic:**
    - Track dismiss count in localStorage (`pwa_install_dismiss_count`)
    - Track last dismiss time in localStorage (`pwa_install_dismiss_time`)
    - Maximum 2 dismissals total, then never show again
    - After first dismissal, re-show after 7 days OR 5 sessions (whichever comes first — for simplicity, implement 7-day delay)
    - If already in standalone mode (`display-mode: standalone`), never show

    **Android/Chrome path:**
    - Listen for `beforeinstallprompt` event
    - Defer the prompt (`event.preventDefault()`, store event)
    - Show banner: "Add SoloStylist to your home screen" with "Install" button
    - "Install" button calls `deferredPrompt.prompt()`
    - On prompt outcome, hide banner

    **iOS Safari path:**
    - Detect iOS via `navigator.userAgent` (check for iPad|iPhone|iPod without MSStream)
    - Show static instruction text: "To install: tap the Share button, then 'Add to Home Screen'"
    - Include a small share icon (from lucide-react: `Share` or `Upload` icon) for visual clarity

    **Visual design:**
    - Fixed position: `bottom-20` (above the bottom nav, below content)
    - `left-4 right-4` for side margins
    - Black background, white text (monochrome theme)
    - Rounded corners: `rounded-xl`
    - Shadow: `shadow-lg`
    - Close/dismiss button: small "X" in top-right corner (`text-white/60`)
    - z-index: 50 (above content, below modals)

    Follow the full implementation from RESEARCH.md Pattern 6, adapting styling to match the monochrome theme.
