**Step 1: Create `src/components/clients/client-search.tsx`** — `'use client'`

A pinned search bar component that queries Supabase via the BROWSER client (not server action — this needs to be fast and debounced).

Props: `onResults: (clients: Client[]) => void`, `onClear: () => void`, `onPendingChange?: (pending: boolean) => void`

Implementation:
- Text input with Search icon (lucide-react), placeholder "Search by name or phone..."
- Styled: sticky top-0, bg-white, z-10, border-b, px-4 py-3
- X button to clear search (resets to full list via onClear)
- `useState<string>` for query text
- `useEffect` with 250ms debounce (`setTimeout` / `clearTimeout` pattern):
  - If query is empty or whitespace, call `onClear()` and return
  - Create browser Supabase client
  - Query: `.from('clients').select('id, first_name, last_name, phone, client_tags(tag_id, tags(name))').or('first_name.ilike.%${query}%,last_name.ilike.%${query}%,phone.ilike.%${query}%').order('first_name').limit(50)`
  - Call `onResults(data ?? [])`
  - Note: RLS automatically filters by owner_user_id so no explicit `.eq('owner_user_id', ...)` needed on browser client queries (RLS uses auth.uid())
- Show subtle loading indicator (opacity change or spinner) while query is in flight
