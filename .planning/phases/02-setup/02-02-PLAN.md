```sql
CREATE TABLE public.services (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  owner_user_id UUID NOT NULL,
  category_id UUID REFERENCES public.service_categories(id) ON DELETE SET NULL,
  name TEXT NOT NULL,
  duration_minutes INTEGER NOT NULL,
  price INTEGER NOT NULL,
  is_active BOOLEAN NOT NULL DEFAULT true,
  deposit_type TEXT NOT NULL DEFAULT 'none' CHECK (deposit_type IN ('none', 'fixed', 'percentage')),
  deposit_value INTEGER NOT NULL DEFAULT 0,
  deposit_required BOOLEAN NOT NULL DEFAULT false,
  created_at TIMESTAMPTZ NOT NULL DEFAULT now(),
  updated_at TIMESTAMPTZ NOT NULL DEFAULT now()
);

CREATE TABLE public.service_categories (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  owner_user_id UUID NOT NULL,
  name TEXT NOT NULL,
  sort_order INTEGER NOT NULL DEFAULT 0,
  created_at TIMESTAMPTZ NOT NULL DEFAULT now(),
  updated_at TIMESTAMPTZ NOT NULL DEFAULT now()
);
```

**Step 1: Create `src/lib/actions/services.ts`.**

```typescript
'use server'
import { createClient } from '@/lib/supabase/server'
```

Define zod schemas:
- `serviceSchema`: name (string min 1), duration_minutes (number int min 5), price (number int min 0 — this is pennies), category_id (string uuid optional/nullable), deposit_type (enum 'none'|'fixed'|'percentage'), deposit_value (number int min 0, default 0), deposit_required (boolean default false)
- `categorySchema`: name (string min 1)

Export these Server Actions:
1. `createService(data)` — validate with serviceSchema, get user via getUser(), insert into services with owner_user_id, revalidatePath('/settings/services'), return `{ success: true }` or `{ error }`
2. `updateService(id: string, data)` — validate, update where id AND owner_user_id match, revalidatePath('/settings/services')
3. `toggleServiceActive(id: string, currentlyActive: boolean)` — update `is_active` to `!currentlyActive` and `updated_at`, revalidatePath('/settings/services')
4. `createCategory(data)` — validate with categorySchema, insert into service_categories with owner_user_id, revalidatePath('/settings/services'), return the created category `{ data: category }` so the sheet can use the new ID immediately

All actions: create server client, getUser() (redirect to /login if no user), validate, mutate, revalidate. Belt-and-suspenders: always include `.eq('owner_user_id', user.id)` on updates even though RLS enforces it.

**Step 1: Create `src/components/services/service-row.tsx`** — `'use client'`

Props: `service: Service`, `onEdit: (service: Service) => void`, `onToggleActive: (id: string, isActive: boolean) => void`

- Row layout: service name (font-semibold), duration (e.g. "45 min", text-sm text-gray-500), price via `formatPrice(service.price)` (font-semibold), category label if present (small coloured text or badge)
- If `service.deposit_required || service.deposit_type !== 'none'`: show a subtle `<Badge variant="outline">Deposit</Badge>` — use the Badge component from shadcn
- Tap row → call `onEdit(service)`
- **Swipe-to-deactivate gesture** using native touch events (NO external library):
  - Track `touchstart`/`touchmove`/`touchend` via refs
  - On `touchstart`: record startX and startY
  - On `touchmove`: calculate deltaX and deltaY. If `|deltaX| > |deltaY|` (horizontal intent), call `e.preventDefault()` to prevent scroll. Apply `transform: translateX(${clampedDelta}px)` to the row element via ref
  - On `touchend`: if leftward deltaX > 80px threshold, call `onToggleActive(service.id, service.is_active)` and animate row back. If threshold not met, animate back to 0.
  - Behind the row (revealed on swipe): a red "Hide" label (or green "Show" if service is inactive)
  - Use `useRef` for touch tracking state (not useState) to avoid re-renders during drag
  - Row element uses `transition-transform` only when not actively being dragged
- If service is inactive: row has `opacity-50` styling

- Empty state: When no services exist, show centered message "No services yet" with subtext "Tap + to add your first service" and a large Plus icon
